1. USER MANAGEMENT & AUTHENTICATION
================================================================================

1.1 Account Registration & Login
    - Secure registration, login, password reset, and invite-based onboarding
    - Invitation tokens sent via email; recipients register or login through
      password-reset workflows
    - Upgrade to modern authentication: OAuth2/JWT via Supabase Auth
    - Two-factor authentication for improved security (client onboarding)

1.2 Role-Based Access Control (RBAC)
    Roles:
      a) Agent
         - Create transactions, confirm wizard data, assign/modify tasks
         - Approve AI communications, upload/delete documents
         - Communicate with all parties, set payment information
         - Assign Elves to their profile
         - Owns personal task templates (unless on a team)
      b) Elf (Transaction Coordinator)
         - Manage assigned transactions, assign/complete tasks
         - Upload/delete documents, approve AI emails
         - Communicate with vendors and clients
         - If standalone, can set payment information
         - CANNOT change master task templates or override agent decisions
      c) Team Lead
         - Control team-wide task templates, modify dependencies/defaults
         - Oversee all team transactions, can act as agent
         - Assign/upload/delete documents, set payment information
         - Assign Elves to their team
         - All activities logged in overview tab
      d) Attorney
         - New RBAC role that uses the same core transaction/document model as
           other internal users
         - Upload legal packets (title commitments, settlement statements,
           affidavits, signed amendments, recording packets)
         - Review title objections, settlement deltas, release timing, and
           recording/disbursement checkpoints for attorney-closing matters
         - Approve or hold attorney-owned sends/releases; every approval writes
           to communication history and audit logs
         - Can use AI-prepared packet indexing, comparisons, and request-note
           drafts
         - CANNOT delegate legal judgment, legal-equivalence decisions, or
           final packet release to AI
      e) Administrator
         - Manage users, set system-wide defaults
         - Configure AI behavior limits (set minimum confidence floor)
         - Access all data, set payment access (on/off)
         - Does NOT control/override team or agent task logic without system prompting
         - Execute communication log exports upon user request
      f) Client (Buyer/Seller in an Agent-Led Transaction)
         - View documents and milestones, ask questions/request info
         - Upload documents (cannot delete, but can flag for deletion)
         - CANNOT edit tasks, delete documents, or see internal notes
         - Can share milestones from their profile
         - Ability to turn on/off notifications
      g) FSBO Customer (Self-Guided Seller)
         - Customer-facing, property-centric workspace for self-managed sales
           and listing prep
         - Upload/view/download documents, review milestones/messages, and ask
           for plain-English AI guidance
         - Can share read-only milestone links, manage notification
           preferences, and flag documents for deletion review
         - Can interact with assigned Velvet Elves support/guide contacts
         - CANNOT see internal workflow logic/notes, edit back-office tasks or
           approvals, or delete documents directly
      h) Vendor
         - Upload documents (if invited); can upload new iterations
           (old documents marked as legacy/outdated)
         - Receive documents, request information
         - Answer requests for task timeline adjustments
         - CANNOT view full document center, see tasks or timelines

1.3 Contact Management
    - Centralized contact directory linked to transactions and vendors
    - Store details for co-agents, loan officers, title reps, attorneys,
      buyers, sellers
    - Functions to update names, emails, phone numbers
    - Vendor contact "card" feature: outbound emails include link/button to

1.4 Brokerage Profiles

1.5 Agent/Team Profiles

    - Integrate internal doc center with task creation/modification
    - Preferred vendors list
    - Buyer and Seller closing checklist templates stored in profile settings

    - Tagged note sections that can be injected into printed closing checklists
    - Seller-specific escrow overage reminders and standard closing guidance
    - Reminders for closed transactions (tax exemptions, reviews)
    - Ability to turn on/off notifications

1.6 New User Experience
    - Overlay tutorial for first-time users (walkthrough)
    - Can be skipped and re-viewed at any time
    - After the first transaction upload, prompt the user to complete their
      profile/setup if required fields or checklist templates are missing

1.7 Client & FSBO Portals
    Standard Client Portal:
      - Ability to turn on/off notifications
      - Share milestones from profile
      - Agent BIO section: "Learn About Your Agent"

    FSBO Customer Workspace:
      - Dashboard sections: Overview, My Properties, Documents,

3. THE WIZARD (AI-DRIVEN TRANSACTION INTAKE ENGINE)
================================================================================

3.1 Wizard Philosophy
    - AI-driven: user uploads documents, AI extracts structured data,
      user confirms/edits, system builds transaction automatically
    - No mandatory manual form-filling upfront (document-first approach)
    - Manual entry fallback when AI extraction fails or documents unavailable
    - Manual entry fields: title company name, representative name, phone, email

3.2 Document Upload (Step 1)
    - Single upload interface for multiple files
    - Global drag-and-drop support anywhere in the authenticated workspace
    - Accepted formats: PDF, JPEG, GIF, DOC, DOCX, and other image types
    - File size limit: up to 20 MB per file
    - Order does not matter
    - Auto-compression for files over 10 MB (Pillow for images,
      Ghostscript for PDFs)
    - Multi-page document splitting: preview step for users to select page
      ranges and split into separate documents with names/metadata
    - Parse DOC/DOCX files (e.g., pre-approval letters) for loan officer
      contact info
    - After drop/upload, AI verifies what the document is, proposes a name,
      asks which transaction it belongs to, and checks whether it should be
      sent for signature

3.3 AI Document Parsing
    - Using switchable AI provider (OpenAI GPT or Claude — see 8.1a)
    - Extract from all uploaded documents:
      Property Data: street address, city, state, zip code
      Transaction Data: representation type, purchase price, financing type,
                        earnest money details, closing mode when applicable
      Timeline Data: contract acceptance date, inspection deadlines,
                     financing/appraisal deadlines, closing date,
                     date-based contingencies
      Party Data: buyer(s), seller(s), listing agent, buyer's agent,
                  loan officer, title company, closing attorney/settlement
                  attorney when present
    - Detect owner-occupied vs. investor/second home purchase (criteria:
      purchase agreement marking, down payment >= 20%)
    - Modify scripts accordingly for investor/second home focus

3.4 Double-Check Mechanism for AI Parsing
    - Two-pass extraction system:
      Pass 1: Read document, produce interpretation
      Pass 2: Re-read using different prompt strategy/extraction template
    - Proceed only if both passes agree with high confidence on key fields
      (date, party, signature status, requested deliverable)
    - If disagreement: request clarification or route to human review
    - Handle edge cases: blurry docs, crooked scans, missing pages
      (halt and request better copy or route to human)

3.5 Address Confirmation
    - AI normalizes address, validates city/state/zip alignment
    - Flag inconsistencies
    - User must confirm address before proceeding

3.6 Purchase Information Validation
    - Identify final purchase price, track changes across counteroffers/amendments
    - Display: all dates from purchase agreements & counter offers,
      final purchase price, professional fees and who pays what
    - Contract acceptance date, closing date, insurance days,
      possession date, inspection days, home warranty info,
      HOA documentation days
    - Verify all documents signed by all parties; flag missing signatures
    - User must explicitly confirm all variable dates, price, fees

3.7 Missing Information Handling
    - System prompts user for missing data
    - If user does not know: AI searches public sources (Google)
      for contact info (loan officers, title company reps)
    - Search based on user-provided context (name + company + location)
    - Display screen showing what AI is searching
    - Results labeled as AI-sourced; user confirmation required
    - No externally sourced data auto-accepted without confirmation

3.8 Wizard Confirmation Page
    - Single confirmation page showing: address, purchase price, closing date,
      closing mode when applicable, all extracted deadlines,
      parties/contact info, financing status, appraisal status (cash only)
    - Actions: edit any field, accept as-is
    - One-time validation (no rescan on edit)
    - Optional "Suggest AI Improvement" button for corrections to improve AI
    - Only after confirmation: create transaction, lock baseline dates,
      generate task list

3.9 Signature Review in Wizard
    - Review documents for signatures
    - If unsigned document found and user has e-signature platform connected:
      offer to send for electronic signature upon transaction finalization
    - Track signature workflow: send, receive, replace original with executed
      copy (old goes to version history), distribute to identified parties
    - If no e-sign connected: inform user and suggest connecting

4. TASK ENGINE (MOST CRITICAL COMPONENT)
================================================================================

4.1 Task Philosophy
    - Use-case driven, dependency driven, mostly automated
    - Continuously optimized by AI
    - NOT static checklists

4.2 Task Templates (Master Library)
    - Import from legacy spreadsheet (50+ tasks from REWORKING TASK DB.xlsx)
    - Structure: Name, Description, Target (who), Dependencies (task or date),
      Float (days before/after), Automation level (Fully automated /
      AI-assisted / Manual), Use-case applicability (Buy-Fin, Buy-Cash,
      Sell-Fin, Sell-Cash)
    - Admin-configurable via UI: edit task definitions, add new tasks,
      configure dependencies
    - Task categories:
      Welcome & Communication: Buyer/Seller/Co-op Agent/Loan Officer Welcome,
                               New Transaction Added
      Documentation & Compliance: Review Documentation, Request/Deliver HOA Docs,
                                  Order/Confirm Title, Buyer Documentation,
                                  Deliver Title/Utility Info, Title Work Completed,
                                  Closing Disclosure Delivered
      Vendor & Contingency: Order/Confirm Home Warranty, Insurance Reminder,
                            Inspection Scheduled/Completed/Response/Negotiated,
                            Appraisal Ordered/Completed, Clear to Close
      Closing & Follow-up: Schedule Closing, Buyer/Seller Closing Info,
                           Agent Closing Info, Closing Gift, Request Referrals,
                           Exemptions/Thank You
      Meta: Closing Date (reference for float calculations)

4.3 Initial Task Generation
    - Upon wizard confirmation: tasks generated strictly from confirmed use case,
      confirmed dates, and dependency logic
    - No AI creativity at this stage

4.4 Dynamic Task Intelligence (AI Agent)
    - As transaction progresses:
      Detect completed steps automatically
      Recommend removal of unnecessary tasks
      Recommend adding missing tasks
      Detect document-based contingencies
      Suggest deadline adjustments (never auto-change)
      When a user creates a manual task, search for similar incomplete tasks
      and offer Add / Combine / Disregard options before saving
    - All additions/removals are RECOMMENDATIONS, not forced
    - Approval rules:
      Individual agents: approve for their transactions
      Team leads: approve for team-wide template changes
      Master task list additions: apply to all Active status transactions
    - Apply new dynamic tasks to: active, incomplete, paused transactions
    - Do NOT add retroactively to completed/closed transactions
    - Preview screen showing affected transactions before confirmation

4.5 Task Recommendation Transparency
    - Added tasks include: reason, source (document/pattern/behavior),
      suggested deadline
    - Removed tasks include: reason, option to restore
    - Nothing disappears silently: removed tasks "sleep"

4.6 Feedback Loop (Learning System)
    - After closing: prompt for quick feedback (useful/unnecessary/missing tasks)
    - AI adapts suggestions per agent or per team over time
    - Compliance tasks never fully removable from all transactions
    - Historical data per user for learning (no GDPR/CCPA compliance needed now)
    - Admin dashboard: track tasks added/removed globally for statistical analysis
    - Task completion rate metrics

4.7 Confidence Threshold System
    - Two-tiered system configurable per team:
      "Ship it" tier: confidence score >= 90% (auto-proceed)
      "I better see it first" tier: confidence score <= 75% (human review)
    - Admin sets global minimum confidence floor
    - Team leads configure higher thresholds but cannot go below admin minimum
    - Task-specific confidence thresholds:
      Lower risk: signature presence, field completion checks
      Higher risk: legal interpretation, clause meaning, obligations
    - UI validates in real-time with warnings; changes logged for audit

4.8 Notification & Reminders
    - Email reminders for due/critical tasks
    - Configurable reminder intervals (day before, due today, past due)
    - Push and in-app notifications
    - SMS reminders when that channel is enabled for the tenant/user
    - Compiled reminder summaries such as "You have 3 transactions with
      deadlines due tomorrow/today"
    - Daily summary email (only when tasks are due, not when there are none)

4.9 Task Sorting & Views
    - Sort tasks by vendor ("vendor carts"): follow up all tasks for one vendor
      in one email across multiple transactions
    - Tasks by status: previous, today, upcoming
    - Highlight tasks with upcoming deadlines

4.10 AI-Generated Closing Checklists
    - Generate 2 closing sheets: 1 for agent, 1 for client
    - Area for agent/team lead to include standard data on all closings
      (escrow accounts, client parties, filing exemptions)
    - Include dates and utility information
    - Changing utilities guide
    - Changing address with USPS and online retailers (Amazon, etc.)
    - Templates are fed from user/team profile settings
    - Separate Buyer and Seller checklist templates
    - Support tagged note injection into the checklist
    - Include seller escrow overage reminders where applicable

4.11 State-Based Task Differences
    - Build state rules around: closing practice (attorney vs title/escrow),
      recording timelines, disbursement timing
    - Workflow differences key off:
      Who runs closing (attorney vs title/escrow company)
      Required attorney involvement in doc prep/settlement
      Recording and funding sequence expectations
      Local customary forms/requirements and compliance steps

5.2 Document Retrieval & Metadata
    - API returning document name, type, ID
    - Rename, archive (soft delete) documents
    - Versioning: track changes, view previous versions
    - Vendors see only their own uploads (unless agent/elf shares)
    - Customer-facing FSBO/client document views should surface status states
      such as Missing, In Progress, Uploaded, Verified, or Complete, with
      role-appropriate actions like Upload New Version or Flag Issue
    - Shared milestone viewers see read-only status cues only; no internal
      workflow details or deletion actions
    - Cross-transaction "All Documents" workspace with AI search by document
      name, buyer, seller, and related transaction

5.3 Document Emailing
    - Choose recipients, subject, body, attachments
    - Templated emails for consistency
    - Direct attachments in emails post-resize (secure links only as fallback)
    - Resend emails with one click from transaction log

5.4 E-Signature Integration
    - Integrate with e-sign providers (DocuSign, HelloSign) directly from
      document center
    - Wizard integration: prompt for unsigned docs post-upload
    - Workflow: send for signature -> user specifies recipients for executed
      copy -> returned signed doc replaces original (old to version history)
      -> distribute to identified parties
    - Named documents (PA, C1, Amend, etc.) auto-sent to all parties/vendors
    - Other documents: notify the responsible internal owner of completion

5.5 AI-Assisted Document Processing
    - AI reads uploaded documents (contracts, PDFs) and auto-extracts key fields
    - Reduce manual data entry, improve accuracy
    - Client/FSBO deletion requests: "flag for deletion" button that notifies
      the responsible internal owner for approval with logging

6. COMMUNICATION ENGINE
================================================================================

6.1 Unified Communication Log (Immutable)
    - All communication in one master log:
      Emails (inbound/outbound), SMS (when enabled), system messages,
      document actions, AI sends, manual edits/sends, and future voice-call
      events
    - Each entry logs: who, what, when, direction, related transaction
    - Immutable record; users can report errors for AI training
    - Search/filtering: by date, party, keyword
    - Efficient indexing (e.g., Elasticsearch)
    - Download: one transaction at a time per user; multi-transaction
      requires admin request via in-app form
    - Retention: 2 years from last user login; quarterly checks for auto-purge

6.2 Email Integration
    - MVP providers: Gmail (primary), Outlook, Apple iCloud
    - Connect email when creating new account
    - Outbound: send via API integration or in-system email
    - Inbound: logged internally, can trigger AI actions
    - IMAP/SMTP: not in MVP, planned for future

6.3 AI Email Responsibilities
    - Auto-respond to vendor/client/FSBO document requests (always CC the
      responsible internal owner, e.g. agent & elf for agent-led files)
    - Auto-respond to client/FSBO factual questions (always CC the responsible
      internal owner):
      Closing date/time, utility info, possession date,
      transaction milestones/status updates
    - Rules:
      Document exists -> send + CC responsible internal owner(s)
      Document missing -> notify responsible internal owner(s) with draft email
      Unsure about request -> notify responsible internal owner(s) with draft email
    - Auto-reply to messages sent back (type out response, ask user to
      modify or send)
    - AI disclaimer at bottom of any AI-sent messages

6.4 AI Draft Safeguards
    - If AI makes assumptions/inferences/interpretations:
      Email drafted but NOT sent
      Responsible human owner(s) receive draft with "Approve / Edit & Send"
      button
      Button links back into system
      All assumptions/inferences/interpretations in bold in draft
    - Side-by-side review UI: left = draft with bolded assumptions,
      right = source data with tooltips
    - Humans always have final say

6.5 AI Email Tone & Guidelines
    - Follow predefined style rules (configurable)
    - Avoid legal advice
    - Avoid certainty when data is inferred
    - Include a disclaimer about verifying information with the responsible
      human owner/support contact (if AI-sent)

6.6 In-App Notifications
    - Notifications for all external communications sent
    - Optional escalations (reminders after 24-48 hours, configurable)
    - Push notifications support

6.7 SMS & Voice Communication Hooks
    - Communication architecture should support future SMS and call-provider
      integration without redesigning the transaction UI
    - Phone icons should support future click-to-call or call-bridge workflows
      where the user can be connected to the selected contact
    - SMS hooks should support reminders, replies, and transaction logging
    - Provider choice can be finalized later, but the data model and logging
      layer should be provider-agnostic

6.8 Vendor Communication (Hybrid Approach)
    - Email asks for date in constrained format ("Reply with: Scheduled: YYYY-MM-DD")
    - AI parses reply and proposes updated task date
    - Vague replies ("soon", "next week"): AI sends clarification email
      automatically or routes to human based on confidence
    - Updates subject to timeline constraint validation

8. AI & AUTOMATION
================================================================================

8.1a Multi-Provider AI Architecture
    - Support both OpenAI GPT and Anthropic Claude as AI providers
    - Provider-agnostic abstraction layer: all AI features (document parsing,
      email automation, task suggestions, wizard logic) must work through a
      unified interface that can route to either provider
    - Admin-configurable provider switching: allow the client to switch the
      active AI provider (OpenAI ↔ Claude) at any time without code changes
    - Provider configuration stored in system settings (not hard-coded)
    - Both providers must support the same input/output contracts so features
      behave consistently regardless of which provider is active
    - Audit logs must record which AI provider was used for each action
    - Rationale: client wants the ability to compare both providers over time
      and switch as the competitive landscape evolves

8.1b AI-Assisted Development
    - Use AI coding agents (Copilot, ChatGPT, Claude) to accelerate
      implementation
    - Guidelines on code quality, documentation, human oversight

8.2 Natural-Language Task Creation
    - Users describe tasks in plain language
    - AI converts to structured task objects with proper due dates/rules
    - Example: "Send inspection contingency reminder three days before
      inspection" -> task with due date and reminder rules

8.3 Predictive Analytics (Later Phase)
    - Predict closing delays, identify at-risk transactions
    - Prioritize tasks based on historical data
    - Hooks built into MVP architecture

8.4 AI Agent - Wizard Enhancements
    - Recognize owner-occupied vs. investor/second home
    - Modify scripts for investor/second home focus
    - Auto-search internet for title/mortgage rep contact info
    - Background search for saved reps: offer updates to info

8.5 Audit Logging for AI
    - Log: document received, what AI extracted, what AI decided,
      why it decided (top drivers), whether human approved/overrode

8.6 Attorney-Role AI Guardrails
    - AI may compare settlement statement versions, extract deadlines, index
      exhibits, summarize communication history, and draft transmittal or
      request language
    - AI must NOT determine legal equivalence, legal position, final packet
      release approval, or same-day disbursement exceptions
    - If confidence is below threshold, a formal/notarized document is
      missing, or legal judgment is required, the task remains human-owned
    - Every attorney approval or override writes to communication history and
      audit logs
